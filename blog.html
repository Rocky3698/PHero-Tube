• var:
    • Has function-level scope.
    • Is hoisted.
    • Can be re-declared and updated.
  • let:
    • Has block-level scope.
    • Is not hoisted.
    • Cannot be re-declared but can be updated.
  • const:
    • Has block-level scope.
    • Is not hoisted.
    • Cannot be re-declared or reassigned (for primitive values), but properties of objects or elements of arrays can be modified.

  • null: Represents intentional absence of value, often used as a placeholder or to indicate that a variable has no value.
  • undefined: Represents unintentional absence of value, typically used to denote uninitialized variables or non-existent properties.

A REST API (Representational State Transfer Application Programming Interface) is a standardized approach to designing web services that use HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources. Key characteristics include:

  • Resource-Based: APIs are centered around resources identified by URIs.
  • Stateless: Requests must contain all necessary information; servers don't store client state between requests.
  • Uniform Interface: Consistent use of standard HTTP methods (GET, POST, PUT, DELETE) and media types (JSON, XML).
  • Client-Server Architecture: Separation of concerns between clients and servers, with clients initiating requests and servers processing them.
  • Cacheability: Responses can be cached by intermediaries to improve performance.
  • Layered System: Allows for the use of intermediaries to improve scalability, security, and performance.

RESTful APIs provide a flexible and scalable way to expose functionality and data over the web, widely used for building web services and integrating systems across different platforms.
